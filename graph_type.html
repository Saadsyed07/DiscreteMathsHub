Graph Theory: Types of Graphs

Overview of common graph types with definitions and examples. This page uses the same layout and styling as other topic pages in DiscreteMathsHub.

What we'll cover: Simple graphs, multigraphs, directed graphs (digraphs), weighted graphs, complete graphs, bipartite graphs and pseudographs — with short examples and use-cases.

Simple Graph

A simple graph is an undirected graph that has no loops (edges that connect a vertex to itself) and no multiple edges between the same pair of vertices.

  • Example: A small friendship network where any two people are connected by at most one friendship link.

Multigraph

A multigraph allows multiple (parallel) edges between the same pair of vertices.

  • Example: Airline routes where more than one flight connects the same two cities.

Directed Graph (Digraph)

In a directed graph, every edge has a direction (an ordered pair). Edges are written as (u → v). Directed graphs model one-way relationships.

  • Example: The "follows" relation on social networks (A follows B but B may not follow A).

Weighted Graph

A weighted graph assigns a numerical weight to each edge (distance, cost, time, capacity).

  • Example: Road network where edges carry distances between cities — used in shortest path algorithms.

Complete Graph

A complete graph (denoted K_n) connects every pair of distinct vertices with exactly one edge.

  • Example: Small fully-connected LAN where every device is directly connected to every other device.

Bipartite Graph

A bipartite graph splits vertices into two disjoint sets such that edges only connect vertices from different sets (no edges within the same part).

  • Example: A job-assignment problem where one partition is workers and the other is tasks.

Pseudograph / Loops

A pseudograph is a graph that may contain loops (edges connecting a vertex to itself) and multiple edges.

  • Example: Network modeling that allows self-transitions or feedback edges.
