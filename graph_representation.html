Graph Representation

Understanding how graphs can be represented in memory and used by algorithms.

Graph representation refers to how we store and visualize graphs in a program. The main types are Adjacency Matrix, Adjacency List, and Incidence Matrix.

1. Adjacency Matrix

A V×V matrix where each cell (i, j) indicates whether an edge exists between vertex i and j.

  • Efficient for dense graphs with many edges
  • Constant-time edge existence checks: O(1)
  • Space complexity: O(V²)
  • Easy to check if graph is complete or symmetric

2. Adjacency List

Stores each vertex and its list of neighbors. Space-efficient for sparse graphs.

  • Space complexity: O(V + E) - much better for sparse graphs
  • Efficient iteration over neighbors
  • Faster for graph traversal algorithms (BFS, DFS)
  • Most common representation in practice

3. Incidence Matrix

Rows represent vertices and columns represent edges, used in algebraic graph theory.

  • Useful in specialized mathematical analysis
  • Space complexity: O(V × E)
  • Good for network flow and electrical circuit analysis
  • Less commonly used than adjacency representations

Quick Questions

Adjacency Matrix is best for dense graphs. It provides constant-time edge lookup O(1) and is suitable when many edges exist. When the number of edges approaches V², a matrix is more efficient than a list.
Adjacency List uses less memory with O(V + E) space complexity. It stores only existing edges and is ideal for sparse graphs where E << V². This is the most common representation in real-world applications.
Incidence matrices are commonly used in algebraic graph theory, network flow problems, and electrical network analysis. They're particularly useful when you need to analyze properties of edges rather than just vertices.
Yes! You can convert between adjacency matrix and adjacency list representations. Matrix to list takes O(V²) time, while list to matrix takes O(V + E) time. Choose the representation based on your algorithm's needs.
