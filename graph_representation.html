Graph Representation

Understanding how graphs can be represented in computer memory and algorithms.

Graph representation refers to how we store and visualize graphs in a program or system. The main types are Adjacency Matrix, Adjacency List, and Incidence Matrix. Each has its own advantages depending on the application.

1. Adjacency Matrix

A 2D matrix (V×V) where each cell (i, j) indicates whether an edge exists between vertex i and j.

  • Efficient for dense graphs.
  • Easy to check edge existence.
  • Space complexity: O(V²).

2. Adjacency List

Stores each vertex and its list of connected vertices (neighbors).

  • Space-efficient for sparse graphs.
  • Easy to iterate through connected vertices.
  • Space complexity: O(V + E).

3. Incidence Matrix

A matrix that shows the relationship between vertices and edges. Rows represent vertices, columns represent edges.

  • Used for algebraic graph theory and network analysis.
